## Project Background

This project was part of a Field Seminar at TU Dresden. Our task was to study the climate conditions in the Alpine region and explore how to utilize modern low-cost sensors for mobile weather data measurement.

Unlike traditional stationary weather stations, we aimed to build a portable mobile mapping device. Ultimately, our team chose to start from scratch—**from hardware procurement and circuit wiring, to embedded C++ programming, and finally to field mapping during an alpine hike**—independently completing the development of this meteorological data acquisition system.

## Embedded Development

The core controller of the system is an **ESP32-C6** development board, which coordinates all external modules:
* **BME280 Environmental Sensor:** Acquires precise temperature, humidity, and air pressure data in real-time via the I2C interface.
* **GPS Module:** Obtains real-time latitude, longitude, altitude (Alt), satellite count, and HDOP precision factor via UART.
* **LCD & MicroSD Modules:** Utilizes the SPI protocol to render data visually on the screen in real-time and persistently log it to the SD card in CSV format.

To ensure the device could operate in the field for extended periods, we used a standard power bank as the power supply and enclosed the entire setup in a transparent protective case. Below is the core data loop collection and persistence code snippet I wrote:

```cpp
// Trigger environmental and spatial data collection every 30 seconds and write to SD card
if (millis() - lastLog > 30000) {
  lastLog = millis();

  // Read BME280 meteorological data
  float T = NAN, H = NAN, P = NAN;
  if (bme.sensorID() != 0) {
    T = bme.readTemperature();
    H = bme.readHumidity();
    P = bme.readPressure() / 100.0;
  }

  // Parse GPS spatial position and altitude
  bool hasFix = gps.location.isValid() && gps.location.age() < 3000;
  double lat = hasFix ? gps.location.lat() : 0;
  double lng = hasFix ? gps.location.lng() : 0;
  bool hasAlt = gps.altitude.isValid();
  double alt = hasAlt ? gps.altitude.meters() : 0;
  
  // Quality factor assessment
  int sats = gps.satellites.isValid() ? (int)gps.satellites.value() : -1;
  double hdop = gps.hdop.isValid() ? gps.hdop.hdop() : -1;

  // Append timestamp and parameters to CSV log
  if (SD.begin(SD_CS_PIN, spiLCD)) {
    appendCSV(T, H, P, hasFix, lat, lng, hasAlt, alt, sats, hdop);
  } else {
    Serial.println("[SD] not ready, skip logging.");
  }
}
```

## Field Mapping & Data Analysis

During the project's three-day itinerary, we took this device deep into the Alps for a hiking mapping expedition. The device operated stably throughout, with the screen providing real-time feedback on current coordinates and weather status, while the SD card silently recorded thousands of data points.

## Conclusion

This interdisciplinary project benefited me immensely. It not only solidified my skills in C++ hardware programming and low-level communication protocols (I2C/SPI/UART), but also gave me firsthand experience of the entire process of how Spatial Data is collected from raw natural environments, cleaned, and ultimately used for visual analysis.
