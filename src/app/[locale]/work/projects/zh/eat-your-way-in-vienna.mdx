## 项目背景

这是我在维也纳工业大学（TU Wien）“制图信息系统（CIS）”课程的期末项目。目标是制作一张交互式地图，不仅能帮用户在市中心找餐厅、咖啡馆和酒吧，还能顺便标出附近的公园和绿地，方便大家买完快餐后找个地方休息。

## 空间数据库 SQL

项目的核心挑战在于底层的数据处理。我将 OpenStreetMap (OSM) 的原始数据导入 PostgreSQL 数据库，并使用 **PostGIS** 扩展来进行空间分析。

因为原始数据非常庞大（包含几百个字段），我编写了 SQL 脚本来提取有用的信息，比如清洗分类、计算公园的面积等。为了让地图在网页上加载得更快，我还专门为数据建立了空间索引：

```sql
-- 建立空间索引，大幅提升前端的地图渲染速度
CREATE INDEX idx_leisure_parks_geom
ON leisure_parks USING GIST (geom);
```


在数据库里把数据清理干净后，使用 **GeoServer** 将它们发布成了地图服务（WMS）。

为了让地图看起来更直观，我编写了 SLD（样式图层描述符）文件，给不同类型的地点配上了专属的小图标（比如咖啡杯代表咖啡馆，酒杯代表酒吧），并将所有图层打包成一个整体，供前端调用。

## 前端交互

网页的前端部分是我用 HTML、CSS、JavaScript 和 **Leaflet** 搭建的。重点实现了以下功能：

* **动态筛选：** 加上了下拉菜单，用户可以过滤，只看自己感兴趣的餐饮或绿地类型。
* **点击查看详情：** 点击地图上的小图标，侧边栏会立刻调用接口，弹出该地点的详细信息（比如菜系、营业时间、是否提供无障碍设施等）。

## 总结

这是一个非常务实的练手项目。它帮我彻底打通了全栈 WebGIS 的开发闭环——从最底层的 PostgreSQL/PostGIS 空间数据清洗，到服务端的 GeoServer 渲染，再到前端地图的交互实现。
